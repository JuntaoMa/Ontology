## Why

当前仓库里的制造设计变更切换成本分析 Demo 已经具备最小展示外壳，但还缺少一套可以让智能体围绕真实业务问题逐步探索本体、形成查询计划、读取实例数据并执行业务函数/动作的工作台能力。要把 Demo 从“展示壳子”推进到“完整流程演示”，需要先沉淀一套面向设计变更场景的专用 skill，并把 schema、查询 IR、业务函数和业务动作统一到同一个可复用入口中。

## What Changes

- 新增一个面向“制造设计变更切换成本分析”场景的专用 skill，用于指导智能体完成 schema 探索、查询规划、Neo4j 查询执行、业务函数调用和完整分析动作编排。
- 新增一套以 JSON 文件为事实源的 schema 资产，覆盖对象类型、关系类型、查询模式、函数契约和动作契约，供 skill 渐进式读取。
- 新增一套面向设计变更场景的查询 IR，支持从业务问题落到锚点对象、多跳路径、过滤条件、函数输入和证据绑定，再编译为 Cypher。
- 新增一组贴合设计变更切换成本分析主链路的查询工具与业务函数，其中查询工具负责变更、批次、库存、到货等事实读取，业务函数负责影响范围归一化、新物料准备周期计算、候选切换点评估、切换成本仿真、候选方案排序和报告组装，并要求计算逻辑与数据查询步骤分离。
- 新增一个 `create_cutover_cost_analysis` 业务动作，用于串联上述函数并输出可展示、可追溯的切换成本分析结果。
- 新增一套以格式化自然语言表达的工具定义与参考文档，用易懂文本描述查询工具、业务函数和业务动作，而不是优先依赖脚本接口。

## Capabilities

### New Capabilities
- `cutover-skill-workbench`: 定义设计变更切换成本分析 skill 的触发方式、工作流、schema 渐进披露方式和输出要求。
- `cutover-query-runtime`: 定义面向设计变更场景的查询 IR、校验规则、Cypher 编译策略和 Neo4j 查询执行行为。
- `cutover-business-functions`: 定义设计变更场景下的核心函数能力、输入输出契约，以及函数逻辑与数据查询职责分离的要求。
- `cutover-analysis-action`: 定义 `create_cutover_cost_analysis` 动作的编排边界、调用顺序、输出结构和失败处理。

### Modified Capabilities

## Impact

- 影响 `.codex/skills/` 下的技能组织方式，将新增一个面向制造设计变更场景的业务化 skill。
- 影响当前 Demo 的后续实现路径，后续智能体链路会以 JSON schema + Neo4j 实例 + QIR 编排为基线。
- 影响设计变更场景的本体表达方式，需要把现有 `data.mjs` 中的 schema、function、action 抽出为可复用的 JSON 资产和自然语言契约。
- 影响后续演示和联调方式，skill 的主要交互将以格式化文本工具定义驱动，同时保留 Neo4j 查询执行和 dry-run 的说明边界。

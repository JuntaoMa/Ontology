## ADDED Requirements

### Requirement: Skill 必须提供面向设计变更场景的工作台式入口
系统 MUST 提供一个专用于制造设计变更切换成本分析场景的 skill。该 skill MUST 指导智能体按固定顺序完成 schema 探索、查询规划、数据读取、业务计算和动作编排，而不是直接跳到自由回答。

#### Scenario: 从业务问题进入工作台流程
- **WHEN** 用户要求分析某个设计变更从哪个批次切换成本最低
- **THEN** skill 指导智能体先识别锚点对象和目标输出，再选择查询工具、业务函数和业务动作

### Requirement: Skill 中的工具定义必须使用格式化自然语言文本
系统 MUST 用格式化自然语言文本定义 skill 中的查询工具、业务函数和业务动作。每个定义 MUST 至少包含名称、用途、必要输入、输出、不负责什么和常见失败。

#### Scenario: 查看某个工具定义
- **WHEN** 智能体读取“计算新物料准备周期”或“定位受影响批次”的工具说明
- **THEN** 该说明以易懂的自然语言模板呈现职责、输入、输出和边界，而不是仅提供脚本命令名

### Requirement: Skill 必须支持 schema 的渐进式披露
系统 MUST 允许智能体先看到 schema 摘要，再按需展开对象、关系、查询模式、函数契约和动作契约的细节，而不是一次性加载全部内容。

#### Scenario: 先看摘要后展开详情
- **WHEN** 智能体首次进入设计变更切换成本分析 skill
- **THEN** skill 先提供 schema 概览，并允许智能体按需继续查看具体对象类型、关系路径和查询模式
